# Restaurant

Проект сайта ресторана, подготовленный для работы с Codex.

## Что внутри

- `AGENTS.md` - инструкции для Codex и других кодовых агентов.
- `SPEC.md` - общий смысл проекта: кто, где, зачем.
- `docs/functional-map.md` - карта пользовательских возможностей.
- `docs/feature-specs/` - спецификации отдельных функций.
- `docs/technical-spec.md` - технические решения и ограничения.
- `docs/visual-rules.md` - правила визуального стиля.
- `docs/user-stories.md` - пользовательские сценарии.
- `TASKS.md` - рабочий план маленькими проверяемыми шагами.
- `index.html` - стартовая страница.
- `styles.css` - базовые стили.
- `script.js` - простая интерактивность.
- `assets/` - папка для изображений и медиа.

## Как открыть

Откройте файл `index.html` в браузере.

Сейчас проект не требует установки зависимостей, сборки или dev-сервера.

## Как работать с Codex

Файл `AGENTS.md` задает правила проекта. При следующих запросах Codex будет использовать его как локальную инструкцию: стиль сайта, структуру файлов, ограничения и проверки.

Рабочий порядок:

1. Сначала обновляем спецификацию.
2. Потом обновляем план.
3. Потом пишем код.
4. После проверки переходим к следующему маленькому шагу.
