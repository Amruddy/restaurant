# Feature Spec: Menu

## Цель

Меню должно показать кухню ресторана, уровень подачи и диапазон цен. Оно должно помогать гостю решить, прийти в ресторан, забронировать стол или перейти к доставке.

Меню не должно перегружать пользователя с первого экрана. Сначала нужно показать основные и популярные позиции, затем дать доступ ко всему меню.

## Роль в пользовательском пути

Меню относится к первому уровню важности вместе с атмосферой и бронированием.

Меню должно работать для нескольких сценариев:

- новый гость понимает кухню и примерный бюджет;
- турист видит современный взгляд на чеченскую кухню;
- постоянный гость быстро находит знакомые блюда;
- пользователь доставки выбирает позиции для заказа;
- гость после просмотра меню переходит к бронированию.

## Категории

В меню должны быть предусмотрены категории:

- популярное или основные позиции;
- салаты;
- супы;
- горячие блюда;
- национальная чеченская кухня;
- европейская кухня;
- азиатская кухня;
- десерты;
- напитки.

Национальная чеченская кухня должна быть отдельной категорией. Она важна для позиционирования "национальная кухня в современном мире" и не должна растворяться внутри общего списка.

## Структура показа

Меню должно иметь два уровня просмотра:

### Первый уровень: основные и популярные позиции

Показывается сначала, чтобы пользователь быстро понял кухню ресторана без перегрузки.

Этот уровень должен включать:

- самые узнаваемые блюда;
- блюда, которые лучше всего передают позиционирование ресторана;
- позиции, которые подходят для первого знакомства;
- блюда, которые могут вести к бронированию или доставке.

### Второй уровень: полное меню

Должен быть доступ ко всем позициям меню.

Полное меню нужно для пользователей, которые хотят:

- изучить весь выбор;
- сравнить категории;
- найти конкретное блюдо;
- заказать доставку;
- понять цены подробнее.

## Данные блюда

Для каждой позиции меню нужны:

- название;
- категория;
- краткое описание;
- цена;
- фотография;
- доступность для доставки.

Цены должны показываться для всех блюд.

Фотографии должны быть у всех блюд. Если реальных фото еще нет, это нужно считать контентным пробелом, а не нормальным состоянием готового сайта.

## Доставка

Доставка должна использовать все позиции меню, если ресторан не задаст ограничения позже.

Если часть блюд нельзя доставлять из-за качества, упаковки или операционных причин, это должно быть описано в `docs/feature-specs/delivery.md`.

Меню должно позволять пользователю понять, какие блюда можно заказать, но детали корзины, оформления и отправки заказа описываются в спецификации доставки.

## Ограничения

- Не показывать блюда без цены в готовом MVP.
- Не использовать пустые заглушки вместо фото в готовом MVP.
- Не перегружать первый экран меню полным списком блюд.
- Не смешивать национальную чеченскую кухню с остальными категориями так, чтобы она теряла значение.
- Не описывать здесь дизайн карточек, анимации, сетки или техническую структуру данных.

## Открытые вопросы

- Какие конкретные блюда входят в категорию популярных.
- Какие блюда входят в национальную чеченскую кухню.
- Какие блюда входят в европейскую кухню.
- Какие блюда входят в азиатскую кухню.
- Какие напитки входят в первый релиз.
- Есть ли блюда, которые нельзя доставлять.
- Какой формат фото будет доступен для всех блюд.

## Тестовое меню

Для разработки MVP можно временно использовать тестовое меню из `docs/sample-menu.md`.

Тестовое меню не является финальным меню ресторана. Его нельзя считать продуктовым решением. Перед публикацией меню должно быть заменено на утвержденное владельцем проекта.
