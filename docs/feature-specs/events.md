# Feature Spec: Events

## Цель

Показать, что ресторан подходит не только для обычного визита, но и для закрытых ужинов и банкетов.

Этот раздел не должен превращать сайт в площадку для всех возможных мероприятий. Ресторан остается обычным рестораном, а события описываются как дополнительный сценарий.

## Роль в пользовательском пути

События относятся ко второму уровню важности.

Главные пути сайта остаются:

- атмосфера;
- меню;
- бронирование.

События должны помогать пользователю понять, можно ли обратиться в ресторан для закрытого ужина или банкета.

## Форматы

Нужно поддержать только основные форматы:

- обычные обеды;
- обычные ужины;
- закрытые ужины;
- банкеты.

Другие форматы не описываются на этом этапе, чтобы не перегружать сайт и не обещать лишние услуги.

## Поведение

Пользователь должен:

- увидеть, что ресторан подходит для закрытого ужина или банкета;
- понять, что детали обсуждаются с администратором;
- перейти к бронированию или контакту.

Отдельная сложная форма события на этом этапе не обязательна. Достаточно пути через общую заявку на бронирование с комментарием или через контакт с рестораном.

## Ограничения

- Не перечислять все возможные типы мероприятий.
- Не обещать конкретную вместимость, если она не уточнена.
- Не обещать отдельные залы, если это не подтверждено.
- Не описывать здесь дизайн блока событий.
- Не описывать техническую обработку заявок.

## Открытые вопросы

- Есть ли точная вместимость для банкетов.
- Есть ли отдельные залы или приватные зоны.
- Нужно ли показывать фото зала или банкетной сервировки.
- Нужно ли позже делать отдельную форму для банкетов.
