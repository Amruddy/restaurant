# Feature Spec: No Separate Visit Scenarios

## Цель

Не добавлять на главную отдельные сценарии посещения, чтобы страница не отвлекала пользователя от главного пути: атмосфера, меню, бронирование.

На текущем этапе сайт не должен продавать специальные сценарии как отдельные направления.

## Роль в пользовательском пути

Это не отдельный пользовательский блок. Основные пути остаются:

- атмосфера;
- меню;
- бронирование.

Сценарии посещения не используются как самостоятельная коммуникация.

## Решение для главной

- Не делать отдельный раздел сценариев посещения.
- Не добавлять отдельные карточки специальных сценариев.
- Не добавлять отдельную форму для специальных сценариев.
- Не использовать эти сценарии как аргументы первого экрана.

## Ограничения

- Не показывать специальные сценарии как отдельные блоки.
- Не обещать отдельные залы или специальные сценарии.
- Не перегружать страницу информацией, которая не ведет к атмосфере, меню или бронированию.
